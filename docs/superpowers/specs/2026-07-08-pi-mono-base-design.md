# pi-mono base monorepo design

## Context

Create a foundation for a npm-workspaces monorepo that publishes pi.dev extensions, modeled on `rpiv-mono` (`/Users/toru/project/EXTERNAL/juicesharp/rpiv-mono/`).

The target repository is `/Users/toru/project/pi-mono`, currently empty except for a minimal `README.md` and `.git`.

## Decisions

- **Approach:** B — root tooling plus one minimal example package.
- **Package scope:** `@rennis23/pi-*`.
- **No build step:** packages publish raw `.ts`, matching `rpiv-mono` and pi package conventions.
- **Single shared config:** one `biome.json`, one `tsconfig.base.json`, one `vitest.config.ts` at the root.
- **Lockstep versioning:** all workspace packages share one version, enforced by `scripts/sync-versions.js`.
- **Husky gating:** `pre-commit` runs formatting/linting/type-checking; `pre-push` runs tests.

## Root tooling details

### package.json

- `name`: `pi-mono`
- `private`: `true`
- `type`: `"module"`
- `workspaces`: `["packages/*"]`
- Scripts:
  - `check`: biome check + tsc --noEmit
  - `test`: vitest run
  - `coverage`: vitest run --coverage
  - `version:patch|minor|major`: bump lockstep version, sync deps, reinstall
  - `release:patch|minor|major`: release script
  - `publish` / `publish:dry`: workspace publish
  - `prepare`: husky
- Dev dependencies: `@biomejs/biome`, `typescript`, `vitest`, `@vitest/coverage-v8`, `@types/node`, `husky`, `shx`.

### tsconfig.base.json

- `target`: `ES2022`
- `module`: `Node16`
- `moduleResolution`: `Node16`
- `strict`: `true`
- `noEmit`: `true`
- Includes `packages/*/**/*.ts`, `test/**/*.ts`, `vitest.config.ts`.

### biome.json

- Preset: `recommended`
- Indent: tabs, line width 120
- Includes `packages/*/**/*.ts`, `test/**/*.ts`, `scripts/**/*.{js,mjs}`, `vitest.config.ts`

### vitest.config.ts

- Include: `packages/*/**/*.test.ts`
- Setup: `./test/setup.ts`
- Coverage provider `v8`, includes `packages/*/**/*.ts`

## Sample package: `@rennis23/pi-hello`

A minimal valid pi extension:

- `package.json` with `pi.extensions: ["./index.ts"]` and peer dependency on `@earendil-works/pi-coding-agent`.
- `index.ts` exports a default extension factory that registers a `/hello` command.
- `hello.test.ts` verifies the command is registered when the factory is invoked with a stub API.

## Scripts

- `scripts/sync-versions.js`: reads every `packages/*/package.json`, verifies lockstep version, updates intra-monorepo `dependencies`/`devDependencies` to `^<version>`.
- `scripts/release.mjs`: checks working tree cleanliness, runs tests with coverage, bumps version, commits, tags, publishes, reinstates `[Unreleased]` changelog sections.

## Out of scope

- Multiple extension packages (to be added later).
- Shared config/util package.
- `.github/workflows`.
- Site/docs package.
- CHANGELOG files for the example package (can be added on first release).

## Success criteria

1. `npm install` succeeds from the repo root.
2. `npm run check` passes (biome + tsc).
3. `npm test` passes.
4. `npm run version:patch` bumps the sample package and root consistently.
5. The sample extension can be loaded conceptually by pi via `pi.extensions` manifest.
